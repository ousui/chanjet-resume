工作总结

针对单机版产品，由最初单纯想使用 bat 脚本进行安装，改为更加可靠的 Install 工具编写脚本安装。

由此选择了开源的 Nsis。

使用 Nsis 顺利将产品打包，并实现了限定系统，注册服务，自动启动，后期又实现了联网在线激活功能。

代码自由完全开源。

现有问题

项目解耦

目前包括 admin, portal, message, 都在一个项目内，导致系统耦合紧密，底层代码稍有改动，就需要重新部署所有的项目，造成了资源的极大浪费。

解决方案：使用 maven 重新解耦，分拆为几个项目，目前正在进行。

运维问题

目前发现比较严重的问题就是，项目代码泄露，如图访问地址：http://t01.exam.chanjet.com/.svn/format

得到的结果为字符串：12，这个是 .svn 目录下的文件，其他文件，如果有有精通 svn 的专家，便可以将代码全部更新，是一个比较严重的漏洞。

其他还有运维管理的服务端中间件版本过低，导致系统中出现一些莫名其妙的BUG，这些都可以通过更新版本来解决。

代码部署过程太过繁琐，需要经过两步 SVN。并且全部需要人工干预。这个目前正在使用 Jenkins 进行解决，实现一键部署。为开发人员节省尽量多的时间。
